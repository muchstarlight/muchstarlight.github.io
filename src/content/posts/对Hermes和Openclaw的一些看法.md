最近在网络上看到了不少推荐 Hermes 和 OpenClaw 的视频，这些视频给人的感觉就是：不去了解这些新技术就落后于时代了。但我实际上手体验之后发现完全不是这样的，甚至这些技术本来就不成熟，我认为完全就是一个中间产物，根本没必要去详细了解。

## 对于 OpenClaw

OpenClaw 这个 AI Agent 想法非常好，确实让 AI Agent 比较像人了，接入即时聊天也促进了这一点。问题是这个东西根本不稳定。我是不太能理解这代码怎么这么冗长的，Vibe Coding 的后果吗？启动起来超级慢，配置工具和本体是一个文件，本身 Node.js 的效率就没那么高，还要完整加载这屎山。

另一个问题就是消耗 token。如果说消耗 token 能够高效地完成一个工作，那没问题，这就是提高生产力。但是这家伙干吃饭不干活啊，消耗大量 token 只为完成一些本来动动手就能完成的工作，这明显是巨大的浪费。至于你说有人拿他来做复杂的工作，那我觉得类似 Claude Code、Codex、OpenCode 这些工具肯定比 OpenClaw 好，毕竟他们的诞生就是为了完成工作。唯一的缺点可能就是不能连聊天工具了，但你真的有那么多活是你不在电脑前就能完成的吗？你真的放心让 AI Agent 开足马力去干吗？

## 对于 Hermes

Hermes 本身也是 Vibe Coding 的产物，但感觉比 OpenClaw 稳定了很多，思路也非常不错，可以总结技能，然后转换成知识。但是这个东西有一个系统性缺陷：只会一味增加知识，而不知道去清理。过时的信息也存着，让 AI 去做错误的事情，反而变笨。

而且你根本不知道 Skill 什么时候创建，虽然内容是透明的，但你可能也无法理清其关系和运作原理，非常难以维护。以及技能会占用上下文的，一大堆技能还没干活能先吃一半上下文（夸张），分散模型注意力，得不偿失啊。

## 总结

目前 AI 正在高速发展，完全没必要实时跟进新内容，除非你是从业者。好用的东西需要时间来检验，我们要选择的不是新概念新技术，而是我们所需要的技术。

这里也稍微谈谈我觉得比较好用的 Agent 吧。Claude Code 算一个，目前公认最强，但我更喜欢 Codex 的跨平台设计，Kimi 的编程工具也不错。 ~~（主要是我买了 Kimi 会员嘻嘻）~~
